# Hawassa Mesob Center Documentation – Consolidation Process & Reference

This file is our master reference for all review, consolidation, and drafting activities. It explains how we work, how we reference this file, how to update it, and rules for continuous progress.

## Purpose

- Track all processes, standards, categories, key milestones, and document the full context and rationale for consolidation.
- Be the single “project logbook” for in-place (or future) AI or user collaborators, providing authoritative reference and continuity.

## How to Reference This File

- In every major task or review prompt, refer to this file as the “Consolidation Process & Reference” document in repo root.
- All process decisions, categorizations, and conventions are defined and iterated here.
- New insights, process changes, file categorizations, and workflow rules should be ADDED here during each major iteration.
- When a file is moved to or produced in `consolidated_documents`, record a short log entry here.

## How and When to Add to This File

- When workflow, file category, or consolidation logic changes, add a section or entry describing what changed and why.
- Whenever a new file is reviewed, added, or finalized in the `consolidated_documents` folder, summarize it here as a mini-log (date, file name, purpose, any open issues).
- At the end of each session/iteration, note outstanding tasks and the current “next prompt” to drive future work.

## Iterative Workflow

1. **Repo Review and Category Discovery**
    - Deeply analyze all files and subfolders to define working categories.
    - Document these categories below and update as new file types or needs emerge.

2. **Category-by-Category Consolidation**
    - For each category, create/maintain a checklist of “candidate files,” open issues, and what consolidation would achieve (e.g., “BoQ analysis – see files X, Y, Z”).
    - For each candidate, track discussion, review, and blocking issues here.

3. **Draft, Review, Finalize**
    - Progress incrementally: each reviewed draft goes to `consolidated_documents` with a log record here.
    - Finalization requires that all iterations, feedback, or unresolved points for that file are also logged here.

4. **Prompt-Driven Iteration**
    - Every AI response ends with a “Next Prompt” tailored to your workflow, referencing open items, this file, and current progress.
    - This file is the anchor for prompting, workflow, and all consolidation logic.
